Introduction

Jekyll is a simple static site generator that can be set as a landing page or an intro for those getting used to web development. You can work with Jekyll environments using a markup language such as HTML and Markdown. Your website can be hosted on Github, avoiding the costs for storing your website on a server. As you learn Jekyll, you will be able to customize your site with original layouts or those located in the Jekyll themes.

Site Creation

Once you have everything in the previous sections installed and working correctly, you can now go ahead and create your default site. When you have jekyll running, you'll be able to see how your website appears before hosting the website on an actual server.

For creating a new site:

  • Open either your Terminal/Command Prompt/GitBash
  • Type the following command "cd" to navigate where you would like to store your website files.
  • Once you have your location determined, type the following command: "Jekyll new site". The "site" can be named to whatever you prefer.
  • The command will execute and install all your default files, you should see something resembling the following figure below:

After running through the previous steps above, a folder will appear where you selected your site to be. You can also view the file path of where the folder appears at the bottom of your terminal. The default options include files such as config.yml and index.md. A sample snapshot for what you should see is provided.

Site Running

You should now be ready to make a test run of what your website looks like. Jekyll provides a default css skin and layout that will generate if you decline changing any features.

To get the site running:

  • Use the command "cd" to navigate to your website's directory
  • Once inside, go ahead and run this command if its the first time you run the website: "bundle exec jekyll serve".

The reason behind running the command "bundle exec jekyll serve" serves a couple of purposes. The command "bundle exec" tells the computer to locate the Gemfile in your current directory. If installed correctly, there should already be one. Then "jekyll serve" will serve you site on the local

Note: You do not have to type "bundle exec jekyll serve" each time you bring the website online. After the first time deployment, you can simply type "jekyll serve" or jekyll s" while you are in your project's directory.

Once you have the previous screen appear on your desktop, you'll need to search the following for where to see your website in development.

  • Locate the "Server Address" section of the terminal. The address shown will display where the website is hosted.
  • Type in the server address on your web browser "localhost:[portnumber]". In the example above, the port number is 4000.

Folder Contents

When you begin to personalize your website, you will be confronted with several folders and files that are part of the configuration. You may open your project with the development studio of your choice. In this example, I am using visual studio code. What will appear will be the screen below:

Below is a brief explanation of what the files and folders include:

  • _posts: This is where your posts will be stored at. They can be written in markdown format if you choose. This directory will be beneficial if you decide on having a blog site.
  • _sites: This directory holds what your final website layout will appear in html.
  • _sass-cache: This holds the css configurations.
  • config.yml: This .yml file acts as a master setting for all variables included in your website.
  • Gemfile: Ruby file that stores the dependencies for the website. This is where you would also store the theme of your website if you choose a pre-made template.

Of course more folders can be added to your website if you so choose. As you continue the tutorial, you will add more directories for different content.

Themes

Note: It is highly recommended that you choose a theme before starting on your project. The reason behind this is some of the layouts you may want to use may not exist in a theme that you decide on. This could result in your page not complying with the CSS and cause loading errors.

Jekyll can have pre-rendered themes that you may want to add to your site. When you download the default settings, the first theme you should receive is the minima theme. To confirm this, you can check your "config.yml" file for your current theme.

If by chance you are wanting to use different thems you can go to RubyGems or Jekyll Themes for other pre-determined themes. Most of the themes will start with something similar to jekyll-theme-xxxx.

To Install a Theme

There are two ways you can install a jekyll theme

  • Grab the theme name from the repository you searched for from RubyGems or Jekyll.
  • Go to your preferred source code editor.
  • Open the file "Gemfile".
  • Add the theme plugin of your choice to your website. It should look like the following:

Another way you can install a theme name is typing in the command of your repository:

"gem install theme-name"

Note: At this point, you will have to shutdown and restart the server for your website due to changing the configuration.

After configuring the gem file

  • Return to the "config.yml" file
  • change the theme name to the one of your choice.

Note; You may also have to alter the layouts you have. Errors will occur is you do not have matching layouts.

Changing Front Matter

What you select for the front matter will be used to display and talk to your website. The front matter acts as like a summary of what the particular page will display. Front matter will always be enclosed with a "---" on th top and bottom section of the variables.

To view the front matter

  • Open your Jekyll site folder in the source editor of your choice. In the example provided, Visual Studio Code is being used.

When editing a page, you should see somehting on any webpage that includes the following:

The figure above holds four common variables for the front matter.

  • layout: What the page matter or skeleton will be
  • title: This is what will be displayed on the tab
  • date: Specifically for posts, but what is the date displayed at the time of creation.
  • categories: Where the static file is stored at and the file path

The front matter is written in YAML and JSON syntax. If you are not familiar with either language, the links provided will lead you to documentation on how to get started.

Any page you create will have to include some type of front matter. For example, the minima theme holds two different layouts when first implemented: page and post. Other themes or custom layouts may only have an option labeled default. Depending on either a chosen theme or a custom theme you will have to test the layout. Each one will take some tweaking before implementing. However, in most cases front matter will have at least the following:

Posts

Note: This section is only needed if you intend to keep your website as a blog. You can skip this section if your website will be used for a different purpose.

Anytime you are wanting to create a blog post with Jekyll content, you will want to store it in _posts. Creating blog posts have certain rules that you must adhere.

To create a blog post

  • Insert a new file into _posts.
  • When creating the file, the name must begin with a date. The date will always following the format YYYY-MM-DD-title-of-post.
  • Give the extension name to be either md for markdown or html for an html document.

You can also define the layout, nut using the post layout will look the best for a blog post in this tutorial.

Note: If you are putting in a title for the post, make sure it contains quotes "" wrapped around it. Otherwise, you will receive mapping value errors:

For drafts

  • Create a new folder called _drafts.
  • Make a new draft for the blog post, but do not give the draft a date.
  • When you are ready to publish the post, update it accordingly with the title that you choose. An example of how to format the draft before publishing and the folder is provided below.
